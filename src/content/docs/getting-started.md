# 入门指南

第一次和 AI 协作时，最重要的不是写出复杂提示词，而是把它当成一个可迭代的工作伙伴。

## 建议流程

1. 说明目标：告诉 AI 你最终想得到什么。
2. 给出上下文：补充限制、受众、材料和当前进度。
3. 要求产物：明确需要清单、代码、草稿、表格还是评审。
4. 分轮推进：先让 AI 给结构，再逐步打磨细节。

## 一个好请求的结构

```text
背景：我正在做一个社区官网。
目标：希望把 Markdown 文档渲染成多页面阅读区。
限制：部署在 GitHub Pages，最好不要依赖后端。
产物：请直接修改项目并说明如何新增文档。
```

## 常见误区

- 只给一句模糊指令，却期待完整结果。
- 一次塞入过多目标，导致反馈难以对齐。
- 没有说明读者是谁，风格会漂移。

把 AI 协作拆成连续的小回合，通常比一次追求完美答案更可靠。
